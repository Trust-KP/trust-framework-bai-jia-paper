TD-10: FRAMEWORK SYSTEM (META) — CANONICAL FORM

------------------------------------------------------------
【CORE STATEMENT】
------------------------------------------------------------

The architecture describes itself.

释定其义，构成其形，度守其界，行成其用，皆归于体构度续。

------------------------------------------------------------
【FOUR FUNCTIONS — META DEFINITION】
------------------------------------------------------------

Function        Definition

Interpretation  category meaning
Framework       structural definition
Regime          control of structure
Realisation     application of structure

------------------------------------------------------------
【PRIMITIVE STRUCTURE — INVARIANT FORM】
------------------------------------------------------------

Each function is defined by the same primitive schema:

Primitive:
- Being
- Structure
- Conditions
- Continuation

------------------------------------------------------------
【INTERPRETATION（释）】
------------------------------------------------------------

Being:
- what is recognised (categories / entities)

Structure:
- relationships between meanings

Conditions:
- rules of interpretation (what counts as valid meaning)

Continuation:
- stability of meaning across contexts / time

------------------------------------------------------------
【FRAMEWORK（构）】
------------------------------------------------------------

Being:
- elements / components defined

Structure:
- organisation of elements

Conditions:
- constraints that hold the structure

Continuation:
- persistence of the structure

------------------------------------------------------------
【REGIME（度）】
------------------------------------------------------------

Being:
- entities under control

Structure:
- control relationships (authority, influence)

Conditions:
- admissibility / enforcement rules

Continuation:
- maintenance of constraint over time

------------------------------------------------------------
【REALISATION（行）】
------------------------------------------------------------

Being:
- actions / events

Structure:
- sequence / process

Conditions:
- execution constraints

Continuation:
- ongoing flow / unfolding

------------------------------------------------------------
【INVARIANCE】
------------------------------------------------------------

- All four functions share identical primitive schema  
- No function introduces external primitives  
- No function collapses into another  

------------------------------------------------------------
【CLOSURE PROPERTY】
------------------------------------------------------------

The system is self-describing:

- Interpretation defines meaning  
- Framework defines structure  
- Regime governs structure  
- Realisation applies structure  

This applies equally to:

- external systems  
- the framework itself  

------------------------------------------------------------
【FINAL STATEMENT】
------------------------------------------------------------

The architecture is internally consistent and self-describing.
